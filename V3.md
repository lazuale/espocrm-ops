# V3

`v3` exists to build a small stable backup/restore tool next to the retained legacy product.

## Mission

`v3` exists to build a small stable backup/restore tool.

## Principles

- minimum code
- maximum backup/restore reliability
- no magic
- no guessing
- no legacy compatibility
- native tools first
- manifest is contract
- verify before trust
- snapshot before destructive restore
- success only after check

## Commands

Current:

- `espops v3 backup verify --manifest <path>`
- `espops v3 backup --scope <dev|prod>`

Planned:

- `espops v3 restore`

Not in first scope:

- `migrate`
- partial backup
- latest lookup
- direct artifact restore/verify

## Backup-Set Contract

- manifest lives in `manifests/`
- backup artifacts live in sibling `db/` and `files/`
- sidecars use `.sha256`
- actual checksum, manifest checksum, and sidecar checksum must match
- `.sql.gz` and `.tar.gz` must be readable
- unsafe tar entries are rejected

## Anti-Goals

- no `resultbridge`
- no `contract/result`
- no shared transport framework
- no acceptance forest
- no oracle harness
- no architecture symmetry
- no compatibility shims

## Cleanup Policy

- delete dead code in separate PRs
- only delete what is demonstrably not used
- do not mix cleanup with feature PRs
- when in doubt, mark the candidate and defer
