# Changelog

Все заметные изменения в этом репозитории фиксируются в этом файле.

Формат вдохновлен Keep a Changelog, но ведется в более свободной и практичной форме для ops-репозитория.

## 2026-04-03

### Добавлено

- Двухконтурная схема `dev` и `prod` на одном `compose.yaml`.
- Скрипты `backup`, `restore`, `migrate-backup`, `stack`, `bootstrap`.
- Скрипт `doctor.sh` для preflight-проверки окружения и конфигурации.
- Скрипт `smoke-test.sh` для изолированной проверки жизненного цикла стека.
- Скрипт `verify-backup.sh` для проверки backup-файлов по SHA-256.
- Скрипт `backup-audit.sh` для проверки свежести, checksum и manifest-полноты последних backup-артефактов.
- Скрипт `backup-catalog.sh` для инвентаризации backup-наборов и быстрой оценки их готовности к restore.
- Скрипт `restore-drill.sh` для изолированной проверки восстановления последних backup во временный контур.
- Скрипт `rollback.sh` для аварийного возврата контура на последний валидный backup-набор.
- Скрипт `docker-cleanup.sh` для безопасного cleanup Docker-хоста с dry-run по умолчанию, без `volume prune`.
- Скрипт `status-report.sh` для краткого text/JSON-отчета по контуру.
- Скрипт `support-bundle.sh` для автоматической сборки диагностического пакета.
- Скрипт `update.sh` для безопасного регламентного обновления контура.
- CI-проверки для shell-скриптов и compose-конфигурации.
- Стартовая инструкция `START.md`.
- Лицензия MIT.
- `manifest`-файлы и checksum-sidecar-файлы для резервных копий.
- `systemd`-юниты для автоматического запуска backup-задач.
- `systemd`-юниты для автоматического запуска backup-аудита.
- `systemd`-юниты для периодического запуска `doctor.sh`.
- Release-памятка `ops/RELEASE.md`.

### Изменено

- Репозиторий очищен от кастомных модулей EspoCRM и собственной сборки образа.
- Восстановление БД и файлов теперь безопаснее: прикладные сервисы автоматически останавливаются перед restore и поднимаются обратно после успешного завершения.
- Восстановление БД и файлов может создавать аварийный backup перед перезаписью состояния через `--snapshot-before-restore`.
- `backup.sh` по умолчанию теперь кратко останавливает прикладные сервисы, чтобы получать более консистентную пару `БД + файлы`; для явного рискованного режима добавлен флаг `--no-stop`.
- Документация переведена и приведена к единому русскоязычному стилю.
- Конфигурация ориентирована на внутренний доступ по IP-адресам внутри корпоративной сети.
- В `compose.yaml` добавлены лимиты ресурсов и ротация логов, управляемые через env-файлы.
- `doctor.sh` теперь дополнительно проверяет рекомендуемые версии Docker/Compose и минимальный свободный объем на файловых системах.
- Backup manifest теперь формируется и в JSON-виде.
- CI теперь дополнительно проверяет корректность `systemd` unit-файлов.
- Добавлена защита от параллельных maintenance-операций через lock-файлы.
- Добавлена автоматическая очистка старых `reports` и `support bundle` по срокам хранения из env-файлов.
- `doctor.sh` получил JSON-режим для автоматизации и вложения в support bundle.
- `status-report.sh` теперь показывает lock-состояние и последние служебные артефакты.
- `smoke-test.sh` и `restore-drill.sh` теперь используют общий helper-слой для временных env-файлов и безопасной очистки каталогов.
- `support-bundle.sh` теперь дополнительно включает текстовый и JSON-каталог backup-наборов.
- `ops/cron.example` синхронизирован с текущим backup/audit/doctor flow и содержит примеры для restore-drill.
- В репозитории появился штатный emergency-rollback flow с pre/post статус-отчетами и автоматическим support bundle при сбое.
- Добавлен отдельный безопасный сценарий cleanup старых Docker-контейнеров, образов, сетей и build cache с dry-run/apply режимами.

### Проверено

- `docker compose config` для `prod` и `dev`.
- `shellcheck` по всем shell-скриптам.
- `bash -n` по всем shell-скриптам.
- Smoke-тест проверки backup-файла через `verify-backup.sh`.
- Реальный restore-drill по временно поднятому контуру и свежему backup.
