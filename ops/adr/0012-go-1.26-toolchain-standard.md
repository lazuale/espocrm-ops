# Go 1.26.2 Is the Repository Toolchain Standard

Historical ADR. Non-authoritative. Repository authority lives in `AGENTS.md`, `AI/spec/*`, generated enforcement under `AI/compiled/*`, `Makefile`, and `.github/workflows/ai-governance.yml`.

## Status

Accepted

## Date

2026-04-18

## Context

After the post-upgrade health audit, the repository was still declaring Go `1.23` even though the active upgraded toolchain and health validation flow had already moved forward to Go `1.26.x`.

That split created two concrete problems:

- the declared module toolchain lagged behind the toolchain actually being used to validate the repository
- `staticcheck` under the upgraded toolchain was already surfacing repository issues that had to be addressed as part of the health remediation

Leaving the repo on the older declared Go line would have kept the local toolchain, CI toolchain, and intended repository standard out of sync.

## Decision

Set the module Go version to `1.26.2` in `go.mod`.

Keep GitHub Actions aligned through `actions/setup-go` with `go-version-file: go.mod`, so CI resolves the same upgraded Go line from the authoritative module declaration.

Do not add a separate `toolchain` directive in this pass. The exact patch-level standard is already declared in `go.mod`, and CI already resolves from that file, so an extra toolchain pin would be redundant rather than necessary for consistency.

Clear the current `staticcheck` failures uncovered under Go `1.26.2` without broadening scope beyond those concrete findings.

## Consequences

- The declared Go version, CI Go version, and intended repository standard now align on Go `1.26.2`.
- The repository no longer presents an outdated Go declaration while using the upgraded toolchain and full health gate.
- Legacy tar regular-file entries remain supported without depending on the deprecated `tar.TypeRegA` constant.

## Rules

- Do not downgrade the declared Go version below the active repository toolchain standard without an explicit new decision.
- Keep CI Go resolution sourced from `go.mod` unless the repository adopts a stronger single source of truth intentionally.
- Treat new analyzer failures introduced by future Go-line upgrades as required health work, not optional cleanup.
